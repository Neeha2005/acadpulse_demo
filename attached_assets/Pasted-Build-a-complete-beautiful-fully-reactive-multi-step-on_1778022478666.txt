REACTIVE BEHAVIOR — critical requirements:

Screen size breakpoints:

Desktop (>768px): content card max-width: 560px, platform cards full width within that, mapping rows horizontal
Tablet (480–768px): same as desktop but max-width: 92vw
Mobile (<480px): platform cards stack vertically (they already are), mapping rows go vertical (group name above, course name below), Continue button full width, step dots shrink to 8px


Step 3 → Step 4 dynamic rendering: read the toggle states saved in React state from Step 3. Only render sections in Step 4 for platforms that are ON. If only Gmail is ON — Step 4 only shows Gmail OAuth section. If all three are ON — show all three sections in order: WhatsApp, Gmail, Classroom
Step 5 dynamic rendering: same logic — only show mapping sections for connected platforms
Progress persistence: on every step completion call POST /onboarding/progress. On page refresh or return visit: call GET /onboarding/status and resume from current_step. Show a banner at top of resumed step: "Welcome back [Name] — picking up where you left off 👋" in glass amber card
Form state persistence in React: use a single top-level state object onboardingData in Onboarding.jsx that accumulates all data across steps. Never lose previously entered data when going back and forward between steps


TECHNICAL REQUIREMENTS:

Single file Onboarding.jsx with sub-components for each step defined inside the same file — no separate files needed
All styles in onboarding.css — import it in Onboarding.jsx
Use useState for current step, form data, toggle states, connection states
Use useEffect for: fetching WhatsApp groups on Step 5 mount, fetching QR code on Step 4 mount, polling WhatsApp connection status every 3 seconds while QR is showing (setInterval cleared on unmount or connection success)
All API calls wrapped in try/catch — never crash the onboarding flow due to a failed API call. Show inline error toasts instead
Toast notification system: a simple Toast.jsx component — fixed bottom-center, glass card style, auto-dismisses after 3 seconds, supports success (green left border) and error (red left border) variants
Custom glass dropdown for semester field — do not use native <select>
Custom toggle switches — do not use any library
