=== ACADPULSE — COMPLETE USER FLOW ===

─── ENTRY & AUTH ───────────────────────

1. User visits the AcadPulse link
2. Login / Signup page is shown

   [ New user ]
   3a. Fill signup form — name, phone number, university, password
   4a. Data saved to DB (users table, password hashed)
   5a. Session created → JWT token issued

   [ Returning user ]
   3b. Fill login form — phone/email + password
   4b. Credentials verified against DB
   5b. Session created → JWT token issued

6. Check: has this user completed onboarding?
   → YES → go to Dashboard
   → NO  → go to Onboarding

─── ONBOARDING ─────────────────────────

Step 1 — Select platforms
   - Tick which to connect: WhatsApp, Gmail, Google Classroom

Step 2 — Connect platforms
   - WhatsApp   → scan QR code (Baileys session)
   - Gmail      → Google OAuth login
   - Classroom  → same Google OAuth token reused

Step 3 — Pick WhatsApp groups to monitor
   - Select subject/course groups
   - Select society/community groups (separate category)

Step 4 — Map groups & classrooms to courses
   - Link each WhatsApp group + Classroom course to a subject name
   - e.g. "OS group + CS301 Classroom = Operating Systems"

Step 5 — Enter timetable
   - Day, time slot, subject name, room number

Step 6 — All onboarding data saved to DB
   → Proceed to Dashboard

─── DASHBOARD ──────────────────────────

1. Backend starts fetching from all connected sources simultaneously:
   - WhatsApp  → Baileys listener (real-time incoming messages)
   - Gmail     → Gmail API poll
   - Classroom → Classroom API poll

2. Every message goes through the AI pipeline:
   - Classifier  → label as Assignment / Quiz / Announcement /
                   Material / Event / Exam Schedule / Noise
   - Noise       → filtered out, never shown
   - LLM extract → pull deadline from message (if applicable)
   - Urgency     → score based on time remaining

3. Processed notifications saved to PostgreSQL

4. Dashboard renders with all tabs populated:
   - Home       → today's urgent items sorted by priority
   - Tasks      → assignments & quizzes, filterable by course/status
   - Materials  → lecture slides, notes, links by course
   - Events     → society announcements, raw message view
   - Chatbot    → ask questions or take actions in Roman Urdu / English

─── ONGOING (after first load) ─────────

- New messages → pipeline runs automatically in background
- Urgency scores update every few hours
- Critical items → desktop popup + WhatsApp self-message alert
- Every morning → daily digest sent to student's own WhatsApp
- Student can manually add notifications anytime
- Semester reset → archives all data, fresh start for new semester
